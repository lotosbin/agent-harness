## 數據流：一次完整的跨鏈轉帳

```
  用戶A (Asset Hub)                    Relay Chain                    用戶B (People Chain)
       │                                  │                                  │
       │  1. 發起轉帳請求                   │                                  │
       │─────────────────────────────────▶│                                  │
       │                                  │                                  │
       │                                  │  2. 鎖定資產 (WithdrawAsset)        │
       │                                  │                                  │
       │  3. XCM 消息                      │                                  │
       │◀─────────────────────────────────│  4. 發送 HRMP 消息到 People Chain  │
       │                                  │─────────────────────────────────▶│
       │                                  │                                  │
       │                                  │              5. 接收並執行 XCM:    │
       │                                  │                 DepositAsset      │
       │                                  │                                  │
       │                                  │  6. 結算完成通知 (XCMP)             │
       │◀─────────────────────────────────│◀─────────────────────────────────│
       │                                  │                                  │
       │  ✅ 轉帳完成                      │                                  │
       │                                  │                                  │
```

## Runtime 升級流程（無需硬分叉）

```
  1. 開發者提出 Runtime 升級提議
           │
           ▼
  2. 治理投票 (代幣持有者投票)
           │
           ├──▶ 否決 ──▶ 流程結束
           │
           ▼
  3. 批准後，新 WASM Runtime 打包進區塊
           │
           ▼
  4. 所有全節點自動從鏈上下載新 WASM
           │
           ▼
  5. Wasmtime 解釋器執行新 Runtime
           │
           ▼
  6. 升級完成（無需停機，無需分叉）
```

## Paseo 與 Polkadot 主網的對齊策略

```
  Polkadot 主網
       │
       │  Parity 發布新功能
       │  (Async Backing / Coretime / etc.)
       │
       ▼
  ┌─────────────────────────────────┐
  │   評估新功能是否適合測試網        │
  └────────────────┬────────────────┘
                   │
                   ▼
  ┌─────────────────────────────────┐
  │   Paseo 社區通過 PAS 提案        │
  │   討論並決定是否引入該功能        │
  └────────────────┬────────────────┘
                   │
                   ▼
  ┌─────────────────────────────────┐
  │   PGT (sudo 多簽) 執行 Runtime  │
  │   升級，在 Paseo 上激活新功能    │
  └────────────────┬────────────────┘
                   │
                   ▼
        開發者在 Paseo 上測試新功能
                   │
                   ▼
        功能穩定後上線 Polkadot 主網
```
