## 二、三层提示工程体系详解

```
┌──────────────────────────────────────────────────────────────┐
│ Layer 3: System Notifications  ← 最高优先级                  │
│ ───────────────────────────────────────────────────────────  │
│ 触发条件：环境事件、任务状态变更、用户中断                    │
│ 示例："检测到文件冲突"、"构建失败"、"超时警告"                │
│ 用途：实时感知环境变化，触发 LLM 调整策略                    │
├──────────────────────────────────────────────────────────────┤
│ Layer 2: System Prompts  ← 角色与行为规范                    │
│ ───────────────────────────────────────────────────────────  │
│ 内容：Agent 角色定义、编码规范、安全策略                      │
│ 示例："你是一个 Code Review Agent"、"遵循 PEP8"、"不执行 rm -rf │
│       未经确认"                                              │
│ 特点：不同模型有不同的 System Prompt 模板                     │
├──────────────────────────────────────────────────────────────┤
│ Layer 1: Tool Descriptions  ← 最小化工具集                  │
│ ───────────────────────────────────────────────────────────  │
│ 设计原则：工具越少越好，降低选择困惑和错误率                  │
│ 内容：工具名称、参数 schema、返回格式、适用场景               │
│ 示例：Tool Use → Tool Result → Tool Progress 完整生命周期     │
└──────────────────────────────────────────────────────────────┘
```

## 五、关键设计原则总结

```
1. 最小化工具集 ────▶ 减少 LLM 选择负担，降低工具调用错误率

2. 模型自适应 ─────▶ 不同模型使用不同内部架构，不强求统一

3. 记忆分层 ──────▶ Session 记忆（单次）+ 跨会话持久化（长期）

4. 权限安全优先 ──▶ 工具执行前权限确认，危险命令拦截

5. 流式输出 ──────▶ AsyncGenerator 逐 token 流式，用户实时感知

6. MCP 扩展 ──────▶ Model Context Protocol 支持第三方工具接入
```
