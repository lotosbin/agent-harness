## 简化版：一目了然

```
想法 ──▶ 头脑风暴 ──▶ 设计 ──▶ 任务规划 ──▶ TDD开发 ──▶ 代码审查 ──▶ 完成
         (提问)       (确认)    (拆分)        (测试)       (检查)      (合并)
```

**一句话理解**：用流程约束 AI 行为，让每一步都有验证。

## 核心原则速记

```
┌────────────────────────────────────┐
│         SUPERPOWERS 铁律            │
├────────────────────────────────────┤
│ 1. 没有失败的测试，就不写代码        │
│ 2. 没有设计文档，就不写代码          │
│ 3. 没有任务规划，就不写代码          │
│ 4. 没有代码审查，就不合并            │
│ 5. 没有验证通过，就不宣布完成        │
└────────────────────────────────────┘
```
