## systematic-debugging：系统化调试

遇到 Bug 时，AI 会自动进入 4 阶段调试流程：

### 4 阶段调试法

| 阶段 | 说明 | 产出 |
|------|------|------|
| **1. 复现** | 找到最小可复现步骤 | 明确的复现命令 |
| **2. 隔离** | 定位问题范围 | 问题边界描述 |
| **3. 追溯** | 找到根本原因 | 根因分析 |
| **4. 验证** | 确认修复有效 | 修复证明 |

### 示例对话

```
你：用户登录后显示的是别人的名字
AI：我正在使用 systematic-debugging 技能...

AI：阶段 1 - 复现
    请提供复现步骤：什么操作后出现这个问题？
你：用户 A 登录后看到的是用户 B 的名字
AI：阶段 2 - 隔离
    正在追踪会话处理逻辑...
    发现问题可能在于 sessions 表的 user_id 映射
...
```

## dispatching-parallel-agents：并行任务分发

当有多个独立任务时，AI 可以同时启动多个子 Agent 并行处理：

### 并行执行示意

```
      ┌─ Task A ─────────┐
      │                   │
主 Agent ─┼─ Task B ─────────┼─ 主 Agent 汇总结果
      │                   │
      └─ Task C ─────────┘
```

### 适用场景

| 场景 | 说明 |
|------|------|
| **多模块开发** | 前后端可以同时开发 |
| **独立功能** | 多个不相关的功能并行 |
| **测试套件** | 并行运行不同测试组 |

### 使用示例

```
你：同时实现评论的增删改查
AI：我正在使用 dispatching-parallel-agents 技能...
    检测到 4 个独立任务：
    - Task 1: 创建评论 (POST)
    - Task 2: 获取评论 (GET)
    - Task 3: 更新评论 (PUT)
    - Task 4: 删除评论 (DELETE)

    正在并行分发任务...
```

## verification-before-completion：完成前验证

在宣布"完成"之前，AI 会自动执行验证检查清单：

### 验证清单

- ✅ 所有测试通过
- ✅ 代码审查问题已修复
- ✅ 文档已更新
- ✅ 没有 lint 错误
- ✅ 类型检查通过

### 示例

```
你：功能开发完了，可以合并了
AI：我正在使用 verification-before-completion 技能...

    验证清单：
    ✅ 测试通过 (47/47)
    ✅ 代码审查通过 (0 严重问题)
    ✅ 文档已更新
    ✅ Lint 检查通过
    ❌ 类型检查失败 - models/comment.py:15

    类型错误需要修复后才能合并。
```

## 团队协作模式

### 统一 AI 工作流

在团队中推广 Superpowers 的关键步骤：

| 步骤 | 行动 | 说明 |
|------|------|------|
| 1 | 选定试点项目 | 选择一个非关键项目尝试 |
| 2 | 统一安装插件 | 确保团队成员使用相同版本 |
| 3 | 建立技能库 | 积累项目特定的调试经验和代码规范 |
| 4 | 持续改进 | 定期回顾 AI 表现，调整工作流 |

### 团队技能库积累

Superpowers 的 Skills 会随使用越来越高效：

- **调试过的 Bug** → 形成调试经验，下次更快定位
- **验证过的方案** → 形成最佳实践，避免重复踩坑
- **审核过的代码** → 形成代码规范，风格统一

**你用得越多，你的 AI Agent 越懂你的项目。**

## 大型代码库管理策略

### 分层技能配置

大型代码库建议采用分层技能配置：

```
project/
├── skills/
│   ├── base/           # 基础技能（所有项目）
│   │   ├── tdd/
│   │   └── code-review/
│   ├── web/            # Web 项目专用
│   │   └── api-design/
│   └── cli/            # CLI 项目专用
│       └── argument-parsing/
└── CLAUDE.md           # 项目说明
```

### Worktree 并行开发

```
main ──────────────────────────────────► main
       │
       ├── feature/user-auth      # 用户认证
       ├── feature/payment        # 支付功能
       └── bugfix/login-error     # 修复登录 Bug
```

每个 worktree 都是独立的 AI 会话，互不干扰。

### 大型项目建议

| 建议 | 说明 |
|------|------|
| **分模块规划** | 大项目拆分为多个子项目，每个独立规划 |
| **统一代码规范** | 通过 CLAUDE.md 固化代码风格 |
| **定期同步** | 定期合并主分支，避免大量冲突 |
| **知识沉淀** | 将 AI 的调试经验整理成文档 |

### 技能组合示例
